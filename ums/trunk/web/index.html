User Modeling Server:

Domain Model:

  • lettura del domain model: html | xml | json
  • modifica del domain model:
    •         url: /domain/concept/:concept_id
        parametri:
         concept_name: il nuovo nome per l'elemento del dominio
         concept_slug: in nuovo slug per l'elemento del dominio
        tipo di richiesta: PUT / POST

        esempio: per modificare l'elemento di dominio con slug 'security' la richiesta sarà:
        /domain/concept/1453

User Model

  • lettura del user model: 
            url: /usermodel/:username.[formato]

       esempio: per leggere lo usermodel dell'utente aaltobelli
    /usermodel/aaltobelli.xml
  • lettura di una parte di user model: 
            url: /usermodel/:username/concept/concept_slug.[formato]

        esempio: per leggere lo usermodel dell'utente aaltobelli
    /usermodel/aaltobelli/concept/cryptology.xml
  • modifica dei livelli di conoscenza del domino: 
            url: /usermodel/:username/concept/concept_slug
        parametri:
         bloom_evaluation:
         bloom_synthesis:
         bloom_analysis:
         bloom_application:
         bloom_understanding:
         bloom_knowledge:

         concept_slug: in nuovo slug per l'elemento del dominio
        tipo di richiesta: PUT / POST

        esempio: per modificare i livelli di conoscenza dell' utente aaltobelli per il concetto con slug 'classical-cryptanalysis' la richiesta sarà:
        /usermodel/aaltobelli/concept/classical-cryptanalysis
